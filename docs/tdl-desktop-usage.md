# TDL Desktop 操作文档

TDL Desktop 是基于 tdl 的 Telegram 下载桌面客户端。它不会发送消息，主要用于消息链接下载、JSON 导入下载和对话浏览下载。所有下载最终由本机的 tdl 执行，文件只保存到你选择的本地目录。

## 快速开始

1. 打开 TDL Desktop，先看顶部状态是否显示 tdl 可用。
2. 在右侧 Telegram 登录区域点击检查，确认是否已经登录。
3. 如果未登录，优先使用连接 Desktop，或者使用 QR 登录。
4. 在左侧选择链接下载、JSON 导入或对话浏览。
5. 填写链接或文件路径，选择本地下载目录。
6. 按需调整并发任务数、单文件线程、DC 资源池和高级配置。
7. 点击开始下载任务，右侧实时下载进度会显示当前状态。
8. 下载结束后到任务历史查看成功、失败或取消记录。

## 界面区域说明

左侧是功能导航。常用入口包括链接下载、JSON 导入、对话浏览、任务历史、操作文档和设置。

中间是当前功能的操作区。链接下载和 JSON 导入会显示输入框、下载目录、参数和命令预览。对话浏览会显示对话列表、消息列表和下载栏。

右侧是状态区。这里会显示任务总数、已完成、已失败、Telegram 登录状态、实时下载进度和最近一行 tdl 输出。

## 首次使用前检查 tdl

顶部状态显示 tdl 可用时，说明应用已经找到了可执行的 tdl。发布版通常会内置 tdl.exe，用户更新过 tdl 后也可能使用用户目录中的新版。

如果显示 tdl 不可用，可以先点击顶部刷新。如果仍不可用，请确认发布包完整、资源目录没有被删除，或者重新安装/解压应用。

设置页的 TDL 区域会显示当前 tdl 的版本和路径。这里还可以设置 tdl 命名空间和 tdl 存储参数。普通用户通常保持默认即可。

## Telegram 登录

首次下载私有内容或浏览对话前，需要让 tdl 登录 Telegram。登录状态显示在右侧 Telegram 登录区域。

### 检查登录状态

1. 点击 Telegram 登录区域右上角的检查。
2. 如果显示已登录，通常可以直接下载。
3. 如果显示尚未登录，请选择连接 Desktop 或 QR 登录。
4. 如果检查超时，请确认网络、代理或 Telegram 连接可用。

### 使用 Telegram Desktop 登录

连接 Desktop 适合已经在本机 Telegram Desktop 登录过账号的情况。

1. 先打开 Telegram Desktop，并确认账号可正常使用。
2. 回到 TDL Desktop 的 Telegram 登录区域。
3. Telegram 目录可以留空，应用会让 tdl 自动查找。
4. 如果 Telegram Desktop 设置了本地密码，在 Desktop 本地密码中填写。
5. 点击连接 Desktop。
6. 等待登录完成后，再点击检查确认状态。

如果自动查找失败，可以手动填写 Telegram Desktop 的数据目录。填写目录时请使用本机绝对路径。

### 使用 QR 登录

QR 登录适合没有 Telegram Desktop 登录态，或者 Desktop 连接失败的情况。

1. 点击 QR 登录。
2. 等待界面显示二维码文本。
3. 使用 Telegram 手机端扫描二维码。
4. 在手机端确认登录。
5. 等待界面显示登录完成。
6. 点击检查确认状态。

登录过程中可以点击取消。取消后如果状态没有立即刷新，可以稍等几秒再点击检查。

### 退出登录

已经登录时，Telegram 登录区域会显示退出登录按钮。退出登录会清理本机 tdl 登录态，不会删除下载文件、任务历史或日志。

如果设置了不同的 tdl 命名空间，退出登录只会清理当前命名空间对应的登录态。

## 下载方式一：链接下载

链接下载适合已经知道 Telegram 消息链接的场景，例如下载某个频道或群组中的一条或多条消息。

1. 在左侧点击链接下载。
2. 在消息链接输入框中粘贴链接，每行一个。
3. 常见格式类似 https://t.me/channel/123。
4. 私有频道或受限内容需要当前 tdl 账号有访问权限。
5. 选择本地下载目录。
6. 按需设置包含后缀、排除后缀、文件名模板等高级参数。
7. 点击开始下载任务。

普通 t.me 链接粘贴后，应用会尝试读取消息文字预览。预览需要 tdl 已登录。预览失败不代表下载一定失败，也不会影响你继续启动下载。

如果要批量下载多条消息，可以一次粘贴多行链接。任务历史会按输入来源记录下载状态。

## 下载方式二：JSON 导入

JSON 导入适合已经有 tdl 可识别的导出 JSON 文件，或者从其他流程导出了消息列表后再统一下载的场景。

1. 在左侧点击 JSON 导入。
2. 在导出文件路径中填写 JSON 文件路径。
3. 每行可以填写一个 JSON 文件路径。
4. 路径示例为 D:\Downloads\result.json。
5. 选择本地下载目录。
6. 按需调整下载参数。
7. 点击开始下载任务。

JSON 文件需要是 tdl download -f 能识别的格式。如果文件路径写错、文件不存在或格式不兼容，任务会失败，并在右侧状态或任务历史中显示错误。

## 下载方式三：对话浏览下载

对话浏览适合不想手动复制消息链接，而是希望从最近消息中选择要下载的内容。

1. 确认 tdl 已登录 Telegram。
2. 在左侧点击对话浏览。
3. 点击对话列表右上角刷新按钮读取对话。
4. 可以用搜索框搜索对话名、用户名或 ID。
5. 选择一个对话。
6. 在顶部选择最近 50、最近 100 或最近 200。
7. 点击刷新消息读取该对话最近消息。
8. 勾选需要下载的消息，也可以点击全选。
9. 在下载目录中填写或选择本地目录。
10. 点击下载。

对话浏览是只读模式。这里没有发送框，不会发送消息，也不会提供账号管理功能。

图片、视频、音频和文件消息会显示媒体类型、大小、时间等信息。图片和部分视频会在进入可见区域后自动加载缩略图。较大的视频或文件可能需要点击加载预览。

缩略图和预览只缓存到本机应用目录，用于本地查看。预览失败不影响实际下载。

## 原始 tdl 参数说明

部分版本或后续版本可能提供原始参数入口。原始参数会直接传给 tdl，适合熟悉 tdl 命令行的高级用户。

如果界面提供原始参数入口，通常只需要填写 tdl 后面的参数，不需要再写 tdl.exe 路径。例如：

```text
download -u https://t.me/channel/123 -d D:\Downloads
```

原始参数模式不会像普通模式那样校验下载目录，tdl 可能写入任意本机路径。普通下载建议优先使用链接下载、JSON 导入或对话浏览。

## 下载目录

本地下载目录必须是绝对路径，例如 D:\Downloads\Telegram。

应用会记住上一次选择的下载目录。目录不存在时，应用会尝试创建。

不要把下载目录设为磁盘根目录，例如 D:\。也不要把下载目录设为应用数据目录。

如果遇到无法创建下载目录、权限不足或路径不可用，请换到用户目录下的普通文件夹，例如下载、桌面或单独创建的文件夹。

## 常用参数推荐

默认参数适合大多数场景：并发任务数 4，单文件线程 4，DC 资源池 8。

网络普通或不确定时，建议先使用默认值。确认稳定后再逐步提高。

网络不稳定、经常失败或疑似被限速时，建议把并发任务数降到 1 到 2，单文件线程降到 1 到 2，DC 资源池降到 1 到 4。

下载单个大文件时，可以尝试把单文件线程提高到 4 到 8。不要一开始就设置很高，过高可能导致连接不稳定。

下载大量小文件时，建议并发任务数 2 到 4，单文件线程 1 到 2。小文件太多时，线程过高通常帮助不大。

如果同时下载多个来源，先保持并发任务数较低，避免 Telegram、网络和磁盘同时压力过大。

## 高级参数说明

### 任务并发 limit

控制同时运行的下载任务数量。数值越大，并发越高，但更容易占用网络、磁盘和 Telegram 连接资源。建议从 4 开始，不稳定时降到 1 或 2。

### 单文件线程 threads

控制单个文件内部的下载线程数量。大文件可以适当提高，小文件较多时不建议过高。

### DC 池 pool

控制 tdl 使用的数据中心连接池大小。网络较好时可以提高，遇到限速、超时或连接不稳定时建议降低。

### 群组下载模式 --group

将同一组媒体按 tdl 的 group 行为处理，适合相册或成组消息。默认开启通常比较适合普通下载。

### 跳过同名文件 --skip-same

目标目录中已有同名且大小相同的文件时跳过下载，适合重复任务和断点场景。默认开启可以减少重复下载。

### 启用断点续传 --continue

尽量继续未完成的下载。适合网络中断、手动取消后继续任务，或者下载大文件时使用。

### 强制重新开始 --restart

忽略已有任务状态重新开始。遇到任务状态异常、续传一直失败或想从头下载时可尝试使用。

一般不要同时依赖 continue 和 restart。想继续就用 continue，想重来就用 restart。

### 倒序导出消息 --desc

导出消息时按倒序处理，常用于从最新消息开始处理。普通下载可以保持关闭。

### 使用 Takeout --takeout

使用 Telegram takeout 会话能力。适合大量数据导出，但可能受到 Telegram 限制，也可能需要等待授权。普通小批量下载通常不需要开启。

### 自动修正后缀 --rewrite-ext

根据媒体类型自动修正文件扩展名，适合扩展名缺失或不准确的媒体文件。

### 包含后缀 include

只下载指定扩展名。多个扩展名用逗号分隔，不需要写点号。例如：

```text
mp4,jpg,png
```

### 排除后缀 exclude

排除指定扩展名。多个扩展名用逗号分隔，不需要写点号。例如：

```text
tmp,part,webp
```

### 保存文件名模板 template

自定义输出文件名。模板能力来自 tdl，适合需要按消息 ID、文件名或日期命名的场景。

模板写错可能导致下载失败或文件名冲突。不熟悉 tdl 模板时建议先留空。

一个常见思路是让文件名包含消息 ID，避免同名文件互相覆盖。实际可用字段以 tdl 支持为准。

## 命令预览

链接下载和 JSON 导入页面底部会显示命令预览。它用于帮助你理解应用将如何调用 tdl。

命令预览中的路径和参数会随着输入内容、下载目录和高级配置变化。

普通用户不需要复制或手动运行这条命令。排查问题时，可以把命令预览和诊断日志一起提供给开发者。

## 实时进度

右侧实时下载进度会优先显示单文件进度。如果 tdl 输出中能识别文件名和百分比，界面会显示每个文件的进度条。

如果只能识别整体百分比，界面会显示整体进度。

如果 tdl 暂时没有输出可识别的百分比，界面会显示运行中或等待 tdl 输出。这不一定代表卡住，可能只是 tdl 当前没有刷新进度。

右侧最新日志会显示最近一行 tdl 输出。下载失败时，这里经常能看到最直接的错误原因。

## 停止、续传和重试

下载过程中可以点击停止当前任务或对话浏览中的取消。应用会终止当前 tdl 进程，并把任务标记为已取消。

取消后已经下载完成的文件通常会保留在下载目录中。未完成的临时文件如何处理取决于 tdl。

如果想继续未完成任务，可以重新发起相同下载，并开启启用断点续传。

如果任务状态异常或续传失败，可以关闭启用断点续传，开启强制重新开始后再试。

如果只是重复下载同一批内容，建议保持跳过同名文件开启。

## 任务历史

任务历史会记录下载来源、下载方式、保存目录、状态和完成时间。

状态包括下载中、已完成、失败和已取消。失败记录会保存错误信息。

任务历史保存在本机用户目录的 .tdl-desktop\history.json 中。

清空历史只会删除历史记录，不会删除已经下载的文件，也不会退出 Telegram 登录。

当前历史列表主要用于查看记录和错误信息。如果要重试，需要回到对应下载页面重新发起任务。

## 设置页

### 常规

可以切换界面语言。

### 日志

可以选择日志目录并生成日志包。日志包用于手动提交给开发者分析，应用不会自动上传日志。

没有自定义日志目录时，日志通常保存在 .tdl-desktop\logs 下。

### TDL

可以查看 tdl 版本和路径。

可以设置 tdl 命名空间。多个命名空间可以隔离登录态和 tdl 数据，普通用户保持 default 即可。

可以设置 tdl 存储参数。例如 type=bolt,path=D:\tdl-data。不了解 tdl 存储配置时建议留空。

### 缓存

清除缓存会清理当前对话列表和已缓存的缩略图。它不会删除下载文件、任务历史或登录状态。

如果对话列表异常、缩略图显示异常或希望释放预览缓存，可以尝试清除缓存。

### TDL Desktop 更新

设置页可以检查 TDL Desktop 更新状态。如果当前版本没有配置更新地址，界面会提示更新地址未配置或更新检查暂未启用。

## 日志与隐私

设置页可以生成诊断日志包。日志包只用于手动提交分析，TDL Desktop 不会自动上传日志。

日志包会尽量脱敏：

- Telegram 用户名
- t.me / telegram.me 链接
- 手机号或长数字 ID
- 本机用户目录
- token、api_hash、authorization、password、passcode 等敏感字段

请在提交日志前自行检查内容，确认没有不希望公开的信息。

## 常见问题

### 顶部显示 tdl 不可用怎么办？

先点击刷新。如果仍不可用，请确认应用发布包完整、resources 目录没有被删除、杀毒软件没有隔离 tdl.exe。重新解压或重新安装通常可以解决发布包损坏问题。

### 检查登录状态失败怎么办？

先确认 tdl 可用，再确认网络和代理可以访问 Telegram。可以先点击检查重试。如果 Desktop 登录失败，尝试 QR 登录。如果 QR 登录失败，取消当前登录流程后重新开始。

### 连接 Desktop 失败怎么办？

确认 Telegram Desktop 已经登录并能正常打开对话。Telegram 目录先留空自动查找。如果 Telegram Desktop 设置了本地密码，需要填写 Desktop 本地密码。如果仍失败，改用 QR 登录。

### QR 二维码没有出现怎么办？

等待几秒看登录日志是否继续输出。如果长时间没有变化，可以点击取消后重新点击 QR 登录。也请确认网络可以连接 Telegram。

### 链接预览失败还能下载吗？

可以尝试下载。链接预览用于显示消息文字，预览失败不一定代表下载失败。私有频道、未登录、网络异常或链接格式特殊都可能导致预览失败。

### 对话列表为空怎么办？

先确认已经登录 Telegram。点击刷新对话。搜索框有内容时可能过滤掉结果，可以清空搜索框。网络慢时读取对话可能需要等待。

### 读不到想要的消息怎么办？

对话浏览默认读取最近 50 条，可以切换到最近 100 或最近 200 后点击刷新消息。如果消息更早，当前界面可能无法直接浏览到，需要使用链接下载或其他 tdl 导出方式。

### 缩略图或视频预览加载失败怎么办？

预览需要 tdl 登录和网络可用。大文件预览可能比较慢。预览失败不影响下载。可以点击单条消息的加载预览重试，也可以到设置页清除缓存后重新打开对话。

### 下载目录报错怎么办？

请使用绝对路径，不要使用磁盘根目录，也不要使用应用数据目录。建议选择 D:\Downloads\Telegram 或用户下载目录下的子文件夹。

### 下载失败但没有明显进度怎么办？

查看右侧最新日志和任务历史中的错误信息。常见原因包括未登录、无权限访问该消息、网络超时、Telegram 限速、JSON 路径错误、文件名冲突或下载目录无权限。

### 下载速度慢或经常中断怎么办？

先降低并发任务数、单文件线程和 DC 资源池。建议从 limit 1 到 2、threads 1 到 2、pool 1 到 4 开始。确认稳定后再慢慢调高。

### 重复下载会覆盖文件吗？

建议保持跳过同名文件开启。文件名模板不合理时可能产生同名冲突。如果需要保留多个版本，可以使用包含消息 ID 的模板。

### 生成日志包后要发给谁？

日志包不会自动上传。只有在你需要反馈问题时，才手动提交给开发者。提交前请自行打开检查，确认没有敏感信息。

## 建议的排查顺序

1. 顶部确认 tdl 可用。
2. Telegram 登录区域点击检查。
3. 确认下载目录是有效绝对路径。
4. 用一条简单链接和默认参数测试。
5. 如果失败，查看右侧最新日志和任务历史错误。
6. 降低 limit、threads 和 pool 后重试。
7. 仍无法解决时，在设置页生成日志包并手动提交给开发者。
