# TDL Desktop 操作文档

TDL Desktop 是基于 tdl 的 Telegram 下载桌面客户端。它不会发送消息，主要用于链接下载、JSON 导入、原始参数执行和对话浏览下载。

## 基本流程

1. 先在首页确认 `tdl` 可用。
2. 如果尚未登录 Telegram，使用 Telegram Desktop 或 QR 登录。
3. 选择下载方式：链接、JSON、原始参数或对话浏览。
4. 选择下载目录并配置高级参数。
5. 点击开始任务。

## 高级参数说明

### 任务并发 limit

控制同时运行的下载任务数量。数值越大，并发越高，但更容易占用网络、磁盘和 Telegram 连接资源。建议从 4 开始。

### 单文件线程 threads

控制单个文件内部的下载线程数量。大文件可以适当提高，小文件较多时不建议过高。

### DC 池 pool

控制 tdl 使用的数据中心连接池大小。网络较好时可以提高，遇到限速或连接不稳定时建议降低。

### 群组模式 --group

将同一组媒体按 tdl 的 group 行为处理，适合相册或成组消息。

### 跳过同名同大小 --skip-same

目标目录中已有同名且大小相同的文件时跳过下载，适合重复任务和断点场景。

### 断点续传 --continue

尽量继续未完成的下载。适合网络中断后继续任务。

### 重启任务 --restart

忽略已有任务状态重新开始。遇到任务状态异常时可尝试使用。

### 倒序导出 --desc

导出消息时按倒序处理，常用于从最新消息开始处理。

### Takeout --takeout

使用 Telegram takeout 会话能力。适合大量数据导出，但可能受 Telegram 限制。

### 自动重写扩展名 --rewrite-ext

根据媒体类型自动修正文件扩展名，适合扩展名缺失或不准确的媒体文件。

### 包含扩展名 include

只下载指定扩展名。多个扩展名用逗号分隔，例如：

```text
mp4,jpg,png
```

### 排除扩展名 exclude

排除指定扩展名。多个扩展名用逗号分隔，例如：

```text
tmp,part,webp
```

### 文件名模板 template

自定义输出文件名。模板能力来自 tdl，常见用法是包含消息 ID、文件名或日期。模板写错可能导致下载失败或文件名冲突。

## 对话浏览下载

对话浏览会读取最近消息列表，选择消息后下载。图片和视频预览会缓存到本机应用目录，仅用于本地查看。

## 日志与隐私

设置页可以生成诊断日志包。日志包只用于手动提交分析，TDL Desktop 不会自动上传日志。

日志包会尽量脱敏：

- Telegram 用户名
- t.me / telegram.me 链接
- 手机号或长数字 ID
- 本机用户目录
- token、api_hash、authorization、password、passcode 等敏感字段

请在提交日志前自行检查内容，确认没有不希望公开的信息。
